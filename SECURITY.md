# Security Policy

## Reporting a vulnerability

Email **shohirevdenis@gmail.com** with the subject line `[genesis-content-os SECURITY]`.

Include:
- A description of the issue and where it lives.
- Steps to reproduce, or a proof-of-concept if you have one.
- Your assessment of impact.

**Do not open a public GitHub issue for security reports.** Public disclosure before a fix puts users at risk.

## Response time

You will get an acknowledgement within **72 hours** of your email. A fix or coordinated disclosure plan follows.

## Scope

Anything in this repository: source code, n8n workflow exports, SQL schema, GitHub Actions, container configs.

Out of scope: third-party services Genesis depends on (Supabase, Doppler, OpenAI, ElevenLabs, HeyGen, Ghost, n8n, Plausible) — report those upstream.

## Credit

If you would like attribution, say so in your report. Reporters are credited in the release notes for the fix unless they ask otherwise.
